# v15 败因定论:正负样本来自两条渲染管线(风格捷径)— 2026-07-23

## 一句话结论

v15(及 v14)训练集的**正样本全部是 `_pad200` 重渲图,负样本全部是旧式原图**——
两条不同渲染管线。模型学会用**渲染风格**区分正负(捷径),而不是密集形态:
"pad200 风格 → 右缘画框;旧风格 → 不画"。这同时解释:

| 现象 | 解释 |
|---|---|
| v15 自家 val mAP≈0.72(虚高) | val 同样是"风格可分"的混合,捷径照样得分 |
| 真实 tip 空背景误火 **19/33 (58%)** | 实盘 tip 窗风格像 pad200(右缘=盘口)→ 不看内容就画框 |
| 真实密集漏检 **0/6** | 从未被迫学习"密集 vs 非密集"的内容差异 |
| v14 同签名(误火 55%) | 同一构造病(4520 张负样本同为旧式原图) |

## v16 数据集规格(核心流程修复,单变量 = 渲染管线统一)

1. **一条管线渲染一切**:所有训练图(正+负)统一走实盘 live 路径:
   `add_mas(全序列) → 取末 200 根(右缘=信号 bar / 盘口) → render_chart`。
   正样本 = 金标框 bar 对齐右缘;负样本 = 同管线渲染的无形态 tip 窗。
2. **负样本三源**(风格全部同构):
   a) 原负样本窗口按同管线重渲(右缘=原窗末 bar);
   b) 真实盘口采集的 tip-empty-ok(collect_v13_tip_previews 持续积累,已 33);
   c) 正样本同币种错位时刻(密集前/后远处)的 tip 窗。
3. **验证与晋升门**(val mAP 永不作晋升依据):
   - 留出的 owner 已审真 tip 金标:应开火命中率 ≫ v12 基线;
   - tip-empty 误火 ≈ 0(v12 的 A′ 后 0/33 是及格线);
   - tip-smoke(实盘同管线)作终审。

   金标扩采继续跑(每脉冲旁路,遵守实盘纪律 8)。
