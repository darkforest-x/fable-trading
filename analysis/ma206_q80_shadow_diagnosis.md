# MA206 q80 影子漏斗诊断

## 结论

- 当前不是“只监控 50 多个币”。本地共有 `401` 个 OKX USDT SWAP 15m 文件；按既定
  宇宙规则排除 `36` 个稳定币、贵金属和股票包装，再排除 `7` 个不足 500 根 K 线的新
  合约后，真实扫描 `358` 个序列。
- 同一前向窗口共有 `18` 个可评分多头候选，q90 通过 `2` 个（`11.1%`），q80 通过
  `4` 个（`22.2%`）。通过率与 top 10% / top 20% 的设计基本一致，LightGBM 没有异常
  卡死；q80 只增加 `2` 个诊断信号。
- 当前更大的问题是运行链路而非阈值。本轮 K 线只覆盖 `2026-07-10 10:30` 至
  `13:30 UTC`，且本机没有 cron/launchd 自动更新任务；第一次诊断时数据落后市场约
  3 小时。
- 主前向扫描器只调用多头 `strict_mask`，没有接入已经存在的 `short_mask`。同时每个
  合约使用 `MIN_GAP_BARS=18`（4.5 小时）去重，因此当前系统本来就不是“密集区每根
  K 线都发信号”，也不是双向交易系统。

## 同窗漏斗

owner 已明确批准 q80 仅作影子诊断。q80 从冻结模型对应的不可变 pre-holdout 验证集重建，
没有读取 holdout，也没有使用前向数据反调阈值。

| 阶段 | 数量 | 说明 |
|---|---:|---|
| 本地 USDT SWAP 文件 | 401 | 增量数据目录中的 15m 合约文件 |
| 满足主线宇宙和最少历史 | 358 | 实际进入 `iter_series(..., min_bars=500)` |
| 前向多头候选 | 18 | 全部限定在同一 `FORWARD_START` 后窗口 |
| q90 通过 | 2 | 阈值 `0.3409333202`，候选通过率 11.1% |
| q80 通过 | 4 | 阈值 `0.3098941866`，候选通过率 22.2% |
| q80 增量 | 2 | 不进入主账本，只进入独立影子账本 |

影子账本首轮共 4 行：1 行已裁决、3 行未裁决。样本远不足以评价收益，尤其不能用唯一
一笔已裁决交易推断 q80 有效。

## 原监控数字为什么误导

`scan_forward_records` 原有 `candidates_seen` 在过滤 `FORWARD_START` 之前累加了所有历史
候选，而 `threshold_signals_seen` 只统计前向窗口后的信号。首轮影子摘要因此显示
`21,302 -> 4`，但这两个数字不在同一时间窗，不能解释过滤率。

本诊断新增的漏斗先把候选限定到 `FORWARD_START` 后，再用同一批 18 个候选比较 q90 和
q80，正确口径是 `18 -> 2 / 4`。

## 交易规则判断

交易规则没有单一的“阈值 bug”，但当前实现和 owner 期望存在两个结构差异：

1. **不是持续实时扫描。** `update_okx` 和前向扫描都是可手工触发的任务，没有 15 分钟
   自动调度；不刷新 K 线就不可能产生新候选。
2. **不是双向密集启动交易。** 主线只扫描多头候选；空头候选函数目前只用于研究数据，
   没有独立模型、独立前向账本和收益终审。

`MIN_DENSE_BARS=5`、expanded 形态阈值、`MIN_GAP_BARS=18` 也会压缩候选，但这些是冻结
规则，不在本轮修改。是否放宽候选规则必须作为单变量实验，不能和 q80 一起改。

## 下一步

1. 在不写主账本的前提下，先完成 `update_okx -> 同窗漏斗 -> q80 影子账本` 的稳定循环，
   并累计至少 24 小时候选率；当前 3 小时样本只能定位管线，不能评价策略。
2. 继续当前因果方向数据集：图片只到信号 bar，标签固定为 long / short / no_trade；等
   E2.1b 自然结束后训练独立方向模型，解决“只有密集框、没有方向”的核心缺口。
3. q80 只保留作召回诊断。它多出的交易必须在独立影子账本中按相同成本和成交假设裁决，
   不能晋升 ACTIVE，也不能与主账本混算。
