# 阶段 2b 报告：判断层（triple-barrier + LightGBM）

**日期**：2026-07-07
**代码**：`src/data/loader.py`、`src/judgment/{candidates,labeling,features,build_dataset,train}.py`
**产出**：`analysis/output/p2b_metrics.json`、`p2b_feature_importance.csv`、`p2b_data_coverage.csv`
**复现**：

```bash
python3 -m src.judgment.build_dataset   # 扫描候选 + 标注 + 特征 → data/judgment_dataset.csv
python3 -m src.judgment.train           # 训练 + val 评估（不触碰 holdout）
python3 -m src.judgment.train --eval-holdout   # holdout 一次性评估（已执行，本报告如实记录）
```

## 一句话结论

**判断层信号统计上真实存在（holdout AUC 0.59，置换检验 p=0.002，稳定优于单特征基线），
但经济上不够用：top-decile 信号扣 0.2% 往返成本后仍为负（-0.11%）。**
根因不在模型而在标签的收益尺度——密集点本身是低波动点，2×ATR 止盈只有约 +0.5%，
成本占掉毛利的 40%。方向有戏，但需要项目所有者决策标签/出场结构再迭代。

## 1. 数据接入

- K 线来源：旧项目 `runs/cache/` 下缓存 CSV，通过软链接 `data/kline_cache` 复用（只读，未拉任何 API）；
- 两种 schema（okx 原始缓存 / gate_ 前缀缓存）在 `src/data/loader.py` 中统一为 OHLCV；
  同一 (交易所, 币种) 的多个缓存文件按 `open_time` 合并去重；
- **15m 覆盖**（明细见 `analysis/output/p2b_data_coverage.csv`）：
  - 319 个可用序列（≥500 根），合计约 225 万根 15m K 线 ≈ **64.3 序列·年**；
  - 只有 `ETH_USDT_SWAP`（okx）有完整 1 年（2025-07 ~ 2026-07）；
  - okx 主力币约 50 个序列有 ~6 个月（2025-12-16 ~ 2026-06-18）；
  - gate 约 112 个序列有 ~3 个月（2026-03-29 ~ 2026-07-04）；
  - **数据缺口**：绝大多数序列不足 6 个月，2025-07 ~ 2025-12 仅 ETH 一条序列，
    时间维度上样本高度集中在 2025-12 之后（尤其 2026-04/05）。

## 2. 规则预筛（候选生成）

移植自旧项目 `tools/build_strict_dense_review_pack.py` 的 strict 模式
（EMA 8/13/21/34/55 密集簇 + EMA144/200 慢线，12 项阈值原样保留：
`fast_spread≤0.28%`、`full_spread≤0.55%`、`fast_slow_gap≤0.35%` 等），**两处修正**：

1. **删除了旧 strict 模式的前视过滤**（`future_favorable / future_ratio` 条件用了未来 72 根数据，
   训练候选池绝不允许）；
2. 同币种 okx/gate 双源重复事件去重（4.5 小时内视为同一事件，保留 okx），序列内沿用旧项目
   18 根最小间隔去重。

**候选数量**：1950 个（100 个币种，2025-07-08 ~ 2026-07-03）。

**Sanity check（对齐"每年约 1500 个候选"）**：本次为 30.3 个/序列·年；ETH 全年序列 104 个。
按旧项目约 100 个币的池子折算 ≈ 3000 个/年，与 1500/年同一数量级、偏多约一倍——
偏多方向符合预期，因为删除了前视的未来收益过滤（旧 strict 用它砍掉过约一半候选）。判定为合理。

## 3. Triple-barrier 标注

- 入场 = 候选确认后**下一根开盘价**；止盈 = 入场 + 2×ATR(14)；止损 = 入场 − 1×ATR(14)；超时 72 根；
- 先触上=1，先触下=0，超时=0（业主确认的简单起步方案）；同根 K 线同时触双障碍按最坏情况计止损；
- `realized_ret` 记录该出场规则下的实际收益，供回测与净收益核算。

**标签分布**（n=1950）：

| 结果 | 数量 | 占比 |
|---|---|---|
| tp（先止盈，label=1） | 803 | 41.2% |
| sl（先止损，label=0） | 1138 | 58.4% |
| sl_ambiguous（同根双触，计0） | 8 | 0.4% |
| timeout（计0） | 1 | 0.05% |

关键观察：**中位出场只有 3 根 K 线（45 分钟）**，超时几乎不存在。密集点 ATR 极小
（TP≈+0.5%、SL≈-0.27%），障碍相对单根 K 线波幅太窄，标签很大程度由入场后 1 小时内的
微观噪声决定。全体样本毛收益均值 +0.06%/笔，扣 0.2% 成本后 **-0.14%/笔**（候选池本身不赚钱，
判断层必须做出足够强的区分才有意义）。

## 5. 切分与评估

**冻结 holdout：`signal_time >= 2026-05-04 00:00 UTC`（最近 2 个月，564 样本）。
本次训练与调参全程未使用；只在最后用 `--eval-holdout` 评估了一次。**
其余按时间 80/20 切 train/val，切分边界做了 73 根（18.25h）purge 防止障碍窗口跨界泄漏：

| 切分 | n | 时间范围 |
|---|---|---|
| train | 1068 | 2025-07-08 ~ 2026-04-22 |
| val | 270 | 2026-04-23 ~ 2026-05-02 |
| holdout（冻结） | 564 | 2026-05-04 ~ 2026-07-03 |

模型：LightGBM 二分类（num_leaves=15、lr=0.05、early stopping on val，最终 16 轮，模型很浅）。
基线：仅 `ma_spread_pct` 单特征逻辑回归。

### 核心指标

| 指标 | val (n=270) | holdout (n=564) |
|---|---|---|
| 正样本率 | 0.452 | 0.394 |
| **ROC-AUC（LightGBM）** | **0.565** | **0.591** |
| ROC-AUC（基线） | 0.468 | 0.523 |
| **PR-AUC（LightGBM）** | **0.503** | **0.491** |
| PR-AUC（基线） | 0.457 | 0.407 |
| 置换检验 p（AUC>0.5） | 0.033 | **0.002** |
| top-decile 毛收益/笔 | +0.13% | +0.09% |
| **top-decile 净收益/笔（扣0.2%）** | **-0.07%** | **-0.11%** |
| top-decile 胜率 | 0.519 | 0.500 |

阈值表（LightGBM）：

| 阈值 | val 信号数 | val precision / recall | holdout 信号数 | holdout precision / recall |
|---|---|---|---|---|
| 0.4 | 154 | 0.500 / 0.631 | 325 | 0.452 / 0.662 |
| 0.5 | 20 | 0.550 / 0.090 | 26 | **0.692** / 0.081 |
| 0.6 | 0 | — | 0 | — |

补充：holdout 上 prob≥0.5 的 26 笔净收益 **+0.03%/笔**（唯一净为正的切片，但 val 同阈值为
-0.05%，26 笔样本不足以下结论）。

### 特征重要性 top10（gain）

`close_vs_ema200`(188) > `ret_4`(121) > `drawdown24`(94) > `close_vs_ema55`(82) >
`pre_range168`(79) > `ma_spread_pct`(76) > `atr_pct`(71) > `spread_chg8`(69) >
`volume_ratio`(69) > `ext_up`(65)。
主导特征是"价格相对慢均线的位置 + 近期动量"，与 P0 发现的 `ma_spread_pct` 波动率代理逻辑一致。

### 与基线对比

LightGBM 在 val/holdout 上 AUC 均领先基线 0.07~0.10，PR-AUC 领先 0.05~0.08，
top-decile 毛收益约为基线 2~3 倍（holdout +0.09% vs +0.03%）——**多特征模型有真实增量**，
单靠 `ma_spread_pct` 在池内基本无区分度（val AUC 0.468 甚至低于 0.5）。

## 7. 已知局限（诚实清单）

1. 数据时间覆盖高度不均：仅 1 条全年序列，样本集中在 2026-04/05；holdout（564）与
   dev（1338）的市场状态可能同质（同一轮行情），p=0.002 的外推性有限；
2. 中位持仓 45 分钟说明标签主要由微观噪声决定，1×ATR 止损在密集（低波动）点位过窄；
3. 往返成本按 0.2% 恒定假设，未含资金费率；也未模拟同一时间多信号的资金约束；
4. okx/gate 跨所去重按 4.5h 窗口近似，可能残留少量重复事件（同向偏乐观）；
5. holdout 仅评估一次，但本报告撰写于评估之后——后续任何改动不得再以 holdout 为反馈。

## 8. 结论与下一步建议

**这条路"有信号、没利润"——值得再迭代一轮，但不值得按当前标签结构直接进阶段 3。**
统计信号真实（模型能从池内挑出相对更可能先触止盈的候选，且显著优于单特征），
失败点是 triple-barrier 的收益尺度太小：TP≈+0.5% 的标签结构下，任何模型都很难覆盖 0.2% 成本。

**需要项目所有者决策的事项**：

1. **障碍尺度**（最关键）：是否改为更宽的障碍（如 TP 4×ATR / SL 2×ATR，或加 ATR 下限
   过滤掉波动过小的候选），把单笔毛利空间抬到成本的 5 倍以上？改动后需重跑全流程
   （属单变量实验，holdout 需换新数据或明确接受复用的过拟合风险）；
2. **数据缺口**：是否投入拉取更长历史（≥1 年、50+ 币）？当前 64 序列·年里 2025 年下半年
   几乎空白，任何"跨市场状态稳定性"结论都做不了；
3. **候选池扩容**：strict 规则去掉前视过滤后仍只产出 ~30 个/序列·年，若放宽到旧项目
   expanded 阈值可将样本量扩大数倍，供模型更充分学习（代价是池内基线胜率更低）；
4. 人工标签（P0 确认的"低回撤"信息）是否作为附加特征做一次单变量实验。
