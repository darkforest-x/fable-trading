## 裁决句（先读）

**链路没断在「缺因子」或「缺出场旋钮」，断在「标签语义不可部署」+「用错裁判」+「regime 不迁移」。**

打通的正确顺序是：

```
重定义触发 bar（因果可判）
  → 小样验证时点对齐（Jaccard / 精度，不是只抬召回）
    → 只在事件窗内学规则（禁全市场裸扫假阳）
      → train 因果 PF + 分边（+可选趋势出作放大器）
        → 预注册后才碰 holdout（第 N 次记账）
          → 影子 → 实盘（纪律 7–12）
```

任何跳过 Phase 0–1、直接「从现有框提特征全市场扫」的路径，今晚已反复证伪。

## 1. 链路卡在哪

### 1.1 一览表（打标 → … → 实盘）

| 环 | 现状（诚实） | 卡点类型 | 证据级 |
|---|---|---|---|
| **打标** | Owner 框多在窗中段（右缘≈50%）；语义是「看完整段密集→启动后再框」的**确认态**；右缘仍满足密集阈值仅 ~1.7% | **语义错位（根）** | 强（`p_owner_label_*`） |
| **触发 bar 定义** | 下游默认把**框右缘**当 tip/入场 bar；与「当下盘口可判」不等价 | **协议未冻结** | 强 |
| **特征 / 因子** | 窄~30 / 富 116：top 仍是 order + spread_chg + 已跌；能分「框时刻 vs 随机」，**不能**变成可部署边 | **瓶颈不在此** | 已排除救援 |
| **规则抽取** | LGBM 披露 → AND 阈值 → **全市场裸扫** → 假阳海或与 spread 同构薄边 | **裁判错位** | 强（E1） |
| **回测（train）** | 最好因果边：spread-short ~1.25；+趋势出 1.32–1.42；分边 short≤1.23 | **发现级薄正 / 不稳** | 强 |
| **Holdout** | #6 v16 tip-replay PF 0.78；#7 A 趋势出 PF≈1.0 | **迁移失败** | 确认级否决 |
| **实盘** | detector=none 诚实空转；三门 30min；ACTIVE 未改 | **无合格挑战者** | 纪律正确 |

### 1.2 机制图（卡点位置）

```
眼睛看见的「该开」 ──hindsight──▶ Owner 框（确认态选点）
                                      │
                                      │ 旧打法：右缘 = tip / 提特征
                                      ▼
                              因果规则 / YOLO tip
                                      │
                    ┌─────────────────┼─────────────────┐
                    ▼                 ▼                 ▼
            全市场裸扫           事件窗内扫          oracle 结算
            （假阳海 / 薄边）    （E3 影子 / 同构）   （PF 5–17，禁止部署）
                    │                 │
                    └────────┬────────┘
                             ▼
                      train 偶过 1.3
                             │
                             ▼
                    holdout ~1.0 / 0.78   ◀── regime 换挡
                             │
                             ▼
                      不可交易 / 空转
```

**卡环排序**：① 打标/触发 bar 语义 ② 入场集合≠眼睛（Jaccard≈0.05）③ regime 不迁移
→ ④ 特征堆叠与出场旋钮**不是**今晚塌因。

## 2. 「打通」的正确定义

### 2.1 三门同开才叫打通（缺一不可）

| 门 | 定义 | 反例（不算打通） |
|---|---|---|
| **A. 因果可扫描** | 触发条件只用信号 bar **及之前**可观测列；任意币、任意时刻可判定 fire/no-fire；无框几何、无「看完启动再框」 | oracle 在框右缘入场；AUC 分框/随机；walk-forward 分数门只在已标样本上滤 |
| **B. Holdout 过线** | **预注册**假设在 ≥2026-05-04 窗，PF@maker ≥**1.3**（主口径），且扣 0.2% 不系统性亏；月度无「单季独撑、其余≈1」的假稳健 | train-only 1.4；E3 按 n 稀疏后 4 月变好；trail 擦线但 baseline 死 |
| **C. 可执行** | 落在 tip/tip−1/tip−2 或显式「确认态」产品语义；新鲜度三门同值；脉冲预算内可算完；不依赖事后窗 | crop-after-box / pad200 中段金标；回看窗模型；detector 需后文才画框 |

### 2.2 明确**不是**成功标准

| 指标 | 为何不算打通 |
|---|---|
| Oracle PF（5–17） | 确认态选点 + 常与标签同向；禁止部署证据 |
| LGBM / AUC（0.95–0.98） | 只说明「标框时刻可分」，默认 hindsight |
| Train PF≥1.3 | 发现级；#7 已证明可整段塌 |
| 召回↑（E1 25%→94%） | 无 Jaccard/精度 → 假阳海 |
| Val mAP / 旧 frozen-F1 | 纪律 12：检测晋升只认 tip 金标 + tip-smoke |
| 出场抬升单独好看 | #7 两档同塌；出场非主因 |

### 2.3 工作定义（写进预注册表头）

> **打通** = 存在一条**冻结的因果触发器 T**（分边）、可选冻结放大器 E（出场/门），满足：
> (1) T 在事件协议下可实时判定；(2) train 发现线过关且含 regime 压力；(3) **新**预注册下
> holdout 第 N 次过线；(4) 影子纸面不破纪律后才谈 promote。

「密集几何有薄 alpha」（emergence ~0.87）**保留为命题未死**；打通的是**可迁移可执行配置**，
不是「形态永远有边」的哲学声明。

## 3. 为什么旧打法打不通（机制）

### 3.1 标签是确认态，下游却当 tip

| 事实 | 机制后果 |
|---|---|
| 框右缘中位在窗宽 ~50%；仍密仅 ~1.7% | 右缘 = **启动已打印**的 bar，不是盘口出生 tip |
| LGBM #1 = `spread_chg8` 扩大 | 特征学的是「已经在散开」，不是「即将出生」 |
| Oracle 有边、因果规则无增量 | 增量锁在**选点集合**，锁不住可规则化条件 |

把「框右缘」喂给 tip 检测 / tip 判断 = 用确认态分布训盘口分布 → #6 判断层**反预测**。

### 3.2 「从框提特征 → 全市场扫」必然假阳或同构薄边

- Owner short 与 `spread_expand`：±18 bar 召回 ~25%、Jaccard ~0.045（规则火点 ≈ 眼睛 5×）。
- E1 放宽对齐：召回 94%，Jaccard **更差**（0.018），`no_tp` PF ~1.14（相对 spread 1.42 **倒退**）。
- 机制：眼睛是**稀疏事后选点**；OHLCV 规则在无事件先验时只能「处处像一点」→ 要么海量假阳稀释边，要么收紧后与已有 spread 触发**同构**（E3/两段式）。

### 3.3 堆因子 / 分边 / 出场救不了错误触发器

| 尝试 | 结果 | 机制 |
|---|---|---|
| 分边 | short 1.13，仍 <1.3 | 必要测量修正，不产生新因果集合 |
| 116 因子 | short 1.23，仍 <1.3；top 语义不变 | 本地 OHLCV 张成空间已饱和 |
| 趋势出 | train +0.17 PF；holdout 与 baseline 同塌 | 放大器；底座无迁移边则同死 |
| E2 atr 门 | 全样本↑，4 月仍 <1 | 抬均值 ≠ 修 regime 裂缝 |
| E3 稀疏 | 4 月↑、他月↓；相对 unsparsed **无边增量** | 脆弱搬家，非结构性 alpha |

### 3.4 Regime：train 利润制度相关

利润堆在 2025H2–2026Q1；2026-04 翻车是预警；holdout 5–7 月 PF~1.0。
**不是结算 bug**（随机入场 holdout 对照仍成立）。同一 A 族再烧 #8 = 重复证伪。

### 3.5 一句话机制总结

> 旧链路用**事后确认标签**训练/拟合**盘口触发器**，再用**全市场密度**结算，
> 在**顺风 regime** 上看到 train 薄正，一换制度就均值回归。
> 缺的不是更多列，是**可部署的触发 bar 协议**和**与协议一致的裁判**。

## 4. 重新设计的打通路径（分阶段 · 单变量）

**总纪律**：每阶段只改一个变量；出口未过不得进入下一阶段；**禁止**把 Phase 3 的
train 过线写成可上线；holdout 仅 Phase 4 且需 owner 预注册批准。

---

### Phase 0 — 重定义「触发 bar」标注协议

**目标**：冻结「什么叫一次可交易触发」，使标签在**当下**可判，切断 crop-after-box /
「看完再框」语义进入下游。

| | |
|---|---|
| **做什么** | 写一页协议（建议落 `docs/` 或本报告附录冻结），明确三选一产品语义并**只选一条主轨**： |
| | **T0a 盘口 tip**：触发 bar = 密集刚够格的出生 bar（或 tip/tip−1/tip−2 窗内首次可判）；图必须右缘=当前、无后文；标签问「此刻会不会开」。 |
| | **T0b 确认态启动**：触发 bar = 满足「已散开 + 方向」的**第一根**可判 bar（显式不是 tip）；允许后文不可见条件下的确认规则。 |
| | **T0c 拒绝混用**：同一金标集禁止同时服务 tip 检测与确认策略。 |
| | 强制字段：`trigger_bar`、`side`、`decidable_at_bar`（布尔）、`protocol_id`、禁止字段进特征：框宽/右缘分数/后文高低点。 |
| **不做什么** | 不再把现有中段框右缘默当 tip；不再 pad200 remap 中段金标冒充 tip 金标；不在本阶段训 YOLO / 扫全市场。 |
| **通过线** | (1) Owner 书面选定 T0a 或 T0b 为主轨；(2) 协议含「当下可判」检查清单（用到的列与窗口写死）；(3) 旧框入库时打标 `legacy_confirm_box`，**默认不进**因果正样本，除非经 Phase 1 回拨映射并通过。 |
| **失败了说明什么** | 若无法写出「不用后文也能判」的检查清单 → 手感本质不可部署，应停主线或只做研究命题，不要进特征工程。 |

**与纪律 12 对齐**：若选 T0a，检测晋升门仍是真 tip 金标 + tip-smoke；若选 T0b，
**不要**用 YOLO tip 检测器当主执行器——应用规则/轻量分类在事件后确认。

---

### Phase 1 — 小样重标或「框右缘回拨 N」专测时点

**目标**：单变量检验「时点」——眼睛的可部署时刻是否存在、落在哪根 bar。

| | |
|---|---|
| **做什么** | **实验 1A（推荐）**：抽 100–200 个历史事件，按 Phase 0 协议重标 `trigger_bar`（两名口径：owner + 协议脚本预标，owner 改判）。 |
| | **实验 1B（便宜对照）**：对现有 short 框，试 `trigger = 右缘 − N`（N∈{0,2,4,8,12,18}），**只**比：与「协议可判第一根」的偏差、以及该 bar 上密集/确认条件成立率——**先不结算全市场 PF**。 |
| | 主度量：时点误差分布、可判率、与旧右缘的 Δbars；辅：小窗内与候选规则的 Jaccard（目标方向 ≥0.12，参照归因预声明）。 |
| **不做什么** | 不在本阶段用 LGBM 提 116 因子；不全市场扫；不申请 holdout；不用召回单指标报喜（E1 教训）。 |
| **通过线** | (1) ≥70% 重标样本存在唯一 `decidable` 触发 bar；(2) 该 bar 上协议条件成立率显著高于右缘（右缘基线 ~1.7% 密）；(3) 1B 若存在 N 使「可判」且与 1A 中位误差 ≤4 bar，可进入 Phase 2 用该映射做**弱监督**，但仍以 1A 为金标。 |
| **失败了说明什么** | 若多数样本「只有看完启动才知道该开」→ 确认态不可因果化；应停止「规则复制眼睛」，改测完全不同命题（或接受无主线 alpha）。若仅 T0b 可判而 T0a 不可 → tip 检测轨应降为旁路采集，主线改确认策略。 |

---

### Phase 2 — 只在事件窗内学规则（禁止全市场裸扫假阳）

**目标**：把学习问题从「全市场哪里像框」改成「**已知事件后**何时/是否触发」。

| | |
|---|---|
| **做什么** | 定义事件先验窗（单变量先冻结一种）：例如 emergence tip 后 wait≤48（已有收集器同构），或「慢锚纠缠段」规则窗。 |
| | 仅在窗内：拟合/搜索**稀疏**确认或启动规则（阈值按**计数/覆盖**校准，禁止按 PF 选阈——E3 纪律）。 |
| | 结算分两行强制并列：**(i) 窗内条件 PF**（研究）**(ii) 部署等价扫描**（事件检测器 + 窗内规则的联合因果扫描）。缺 (ii) 不得宣称可部署。 |
| | 重叠表：精度 / Jaccard / 召回 vs Phase 1 金标；**Jaccard 与精度优先于召回**。 |
| **不做什么** | E1 式全市场宽 AND；把窗内 oracle 高 PF 当成功；与 E3 thr 打包调参；引入未批准的新数据源（资金费等）——若要加，另开单变量阶段。 |
| **通过线** | (1) 相对 E1 宽 R3：精度明显↑且 Jaccard≥0.12（或书面下调目标并说明为何仍可部署）；(2) 部署等价扫描 PF@maker 有相对「仅事件 tip / 仅旧 spread」的**清晰增量**（建议 ΔPF≥0.05 且绝对值逼近 1.3）；(3) n 落入预声明带（防假阳与防过稀）。 |
| **失败了说明什么** | 窗内能对齐眼睛但部署扫描无增量 → 边来自事件定义本身（回退测 tip base rate），不是新入场 alpha。窗内也对不齐 → Phase 0/1 协议仍错，回退，不堆特征。 |

**E3/两段式教训写入本阶段门禁**：若最终规则与 `spread_expand` 同 thr 同构，记「无独立机制」，**不得**当 Phase 2 通过。

---

### Phase 3 — Train 因果 PF + 分边 +（可选）趋势出

**目标**：在**冻结的入场 T** 上做发现级交易结算；出场只作对照放大器。

| | |
|---|---|
| **做什么** | 固定 Phase 2 出口的 T；**强制分边** long/short 分表。 |
| | **双报出场**（E4 纪律）：baseline TP5/SL2 **与** `no_tp`/`trail` 同表；禁止只报趋势出。 |
| | Regime 压力（发现级，不作门优化）：至少报告 2026-04 类翻车月；可选单变量门（atr/BTC）须预声明，且不得「只删坏月」。 |
| | 成本双报：maker 0.06% + legacy 0.2%。 |
| **不做什么** | 同一轮既改入场又改出场又加门；rich 特征再扩；用 AUC/oracle 代替因果 PF；对已证伪 A（unsparsed spread+趋势出）微调后声称新挑战者。 |
| **通过线（发现级）** | 主边（预期 short 或协议指定边）PF@maker ≥1.3；baseline 与趋势出**同时**讲得通（若仅 trail≥1.3 而 baseline≪1.3 → **不通过**，记作出场幻觉）；≥1 个历史压力月 PF 不崩到「靠删样本才好看」；与 Phase 2 重叠度量仍达标。 |
| **失败了说明什么** | 入场对齐了仍 train 不过线 → 眼睛集合无扣成本 alpha（或成本假设下不可交易）。仅趋势出过线 → 回到 #7 风险类，不配 holdout。压力月全靠稀疏/门修补 → 同 E2/E3，不配预注册。 |

---

### Phase 4 — 预注册后 Holdout（强调次数）

**目标**：唯一确认级裁判；消耗必须可见、不可逆记账。

| | |
|---|---|
| **做什么** | 提交预注册卡（建议模板见 §7）：假设名、与旧 A/E1/E2/E3 的**单变量差异**、冻结 T/E/门、宇宙、成本、成功线、失败后处理。 |
| | Owner **显式批准**「这是 holdout 第 **N** 次」（当前下一枪为 **#8**）。 |
| | 只跑预注册范围；报告写「第 N 次」；结果无论正负入库。 |
| **不做什么** | 未过 Phase 3 发现线就申请；同一配置族「再看一眼」；一次批多个出场/多个 thr 网格；测 oracle。 |
| **通过线** | 预注册主口径 holdout PF@maker ≥1.3；净收益叙事与 maker/0.2% 一致；月度无「全≈1.0」；样本量子足以否决（#7 量级：千笔以上更佳）。 |
| **失败了说明什么** | 再次证明该**配置**不迁移——放弃配置，不放弃「记账纪律」。连续失败 → 提高 Phase 0–2 门槛或收摊换命题，而不是加速烧 holdout。 |

**硬前置（来自归因报告，重申）**：新假设须同时 (a) train 过线 (b) 写清与已否决族的单变量差异 (c) 有 regime 压力陈述。否则拒绝进入 #8。

---

### Phase 5 — 影子 → 实盘

**目标**：把确认级边变成可执行管道，不破坏实盘纪律。

| | |
|---|---|
| **做什么** | 影子：forward 纸面或等价日志，**不**自动 promote；核对新鲜度三门同值、脉冲耗时、触发 bar 与协议一致。 |
| | 实盘：仅 owner 逐次授权；仓位/API/kill 开关不代劳。 |
| | 若主轨是 T0a：检测器须过 tip-smoke；无验证检测器时保持 detector=none。 |
| | 若主轨是 T0b：执行器走确认规则路径，**不要**假装 tip 检测已复活。 |
| **不做什么** | 未过 holdout 开真仓；清 forward_log；改 ACTIVE 未点头；往脉冲塞实验扫描（纪律 8）；只改一道新鲜度门。 |
| **通过线** | 影子期约定笔数（如新鲜 50–100）行为与回测同符号；无结构性挡 tip/超时；三门与延迟预算表一致。 |
| **失败了说明什么** | 纸面边在活数据消失 → 执行/延迟/分布漂移问题，回 Phase 2–3 诊断，不靠调阈值续命。 |

## 5. 哪些技术用得上；哪些是弯路

### 5.1 按阶段的正当用途

## 6. 时间与风险诚实评估

### 6.1 量级（日历周，一人主攻研究轨）

| 阶段 | 乐观 | 基准 | 悲观 | 主要风险 |
|---|---|---|---|---|
| Phase 0 | 2–3 天 | **1 周** | 1.5 周 | Owner 不愿在 tip vs 确认态上做产品选择 |
| Phase 1 | 1 周 | **1.5–2 周** | 3 周 | 重标显示不可因果化 → 主线停 |
| Phase 2 | 1 周 | **2 周** | 3 周 | 再落入假阳或与 spread 同构 |
| Phase 3 | 3–5 天 | **1 周** | 1.5 周 | train 过线但压力月碎 |
| Phase 4 | 1 天跑数 | **等批准 + 1 天** | — | **再次证伪（高先验）** |
| Phase 5 | 1–2 周影子 | **2–4 周** | 更长 | 活数据漂移 / 执行摩擦 |

**端到端基准：约 6–10 周**到「配不配申请 #8」的清晰答案；再到可交易实盘另加影子周。
试图 1–2 周「从现有框直接挖通」= 重复今晚路径。

### 6.2 风险诚实声明

1. **Holdout 再证伪是默认先验，不是意外**——#7 已否一强挑战者；新协议仍可能无迁移边。  
2. **打通失败的有价值结局**：证明「确认态手感不可部署」或「tip 出生无扣成本 alpha」——应停烧 holdout，而不是换旋钮。  
3. **时间主要耗在 Phase 0–1（人）**，不是 GPU；YOLO 不是关键路径。  
4. **样本稀缺**：真 tip 金标与协议重标都贵；用自动预标冒充 owner 真值已有先例事故（v16 empty 预标）。  
5. **本报告未跑新数、未动 holdout**；数值均引用已发表报告。  
6. 若 Phase 1 失败，最省的决策是**收摊换命题**，不是加快 Phase 3–4。
