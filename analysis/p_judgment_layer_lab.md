## 1. 目标与成功标准

- **链路**:v16 检测(因果盘口)→ 方向规则 → **判断层过滤(本文件主攻)** → TP5/SL2 → 交易。
- **判断层的活 = 3 个决策**:①选点(top-decile)②方向(long/short)③regime(何时活跃)。
- **成功** = walk-forward 三期(尤其最近的 period3)top-decile PF@maker **稳健 ≥1.3**,
  不靠删样本、不靠单期独撑。达标 → 写预注册卡 → holdout#8。

**关键约定(避免自欺)**
- 只看**因果 PF**,不看 AUC/oracle(它们默认 hindsight)。
- 每个"救活"必须跨三期成立;单期漂亮(如 period1 2.1)不算通。
- 特征富堆已被证伪多次(28/116/130);新特征要有**机制假设**,不是撒网。

### IT-14 · 冻结 COCO 视觉方向预检 —— 🔴 红灯(不开 3060 双检测器)
- **假设**:tip 窗像素 gestalt 携带表格没有的方向边。

### IT-09 · 多头也给完整待遇 + 多空互补 —— ✅✅ 关键转向
- **假设(owner)**:v16 多空都检测,不该只做空;可能空头管熊月、多头管牛月,合并更稳。
- **改动**:多头(上方)配 9 个多头质量特征+多头判断层;空头(下方)同样;各做各边。
- **结果(top20)**:空头 1.67/1.26/0.74;多头 0.84/0.99/**1.09**。**互补**:空头前两期、多头 period3。
- **机制**:方向该随 regime 切(熊做空/回暖做多);IT-08 的"间歇"其实是"该换边没换"。
- **注意**:天真两边都做会稀释(合并 1.30/0.94/0.82,每期总有一边拖后腿)。
- **→ IT-10:causal regime 选边 + 各自判断层(IT-05 失败是因为当时没独立判断层,现在有了)。**

### IT-14 · 视觉方向预检(owner 双检测器想法的省钱门)—— 🔴 未过(第4条证据)
- **假设(owner)**:把 v16 拆成多头/空头两个检测器,让 CNN 看盘口图的视觉 gestalt 学方向。
- **先摸清**:owner 2026-07-23 已手标 2525 框、富特征因果规则 short 1.23/long 0.94(<1.3,
  已成 learning);真正没试的只有"CNN 视觉"。
- **省钱预检(Mac,不烧 3060)**:4012 个 tip 渲染成盘口图(框在最右,无未来)→ yolo11n(COCO)
  256维嵌入 → LGBM 学方向 → walk-forward,和同批表格 130 特征对照。
- **结果**:视觉 AUC 0.46/0.507/0.475(≈硬币,最近期<0.5);方向 PF 1.07/1.10/**0.65**。
  表格同批 AUC ~0.5。**视觉没有携带表格漏掉的方向信息。**
- **注意**:冻结 COCO 底座非图表微调,负面"强但非终局";但 256维富视觉特征都到不了 0.53,
  加上 IT-02/owner-side/IT-12-13 已是**第 4 条独立证据**说方向不可 causal 恢复。
- **→ 3060 全量训练两套检测器的先验已很低;是否仍要打这一枪 = owner 决策。**

### IT-13 · fade 突破入场(均值回归)—— 🔴 淘汰(~1.0,互补翻转依旧)
- **假设**:IT-12 续势全亏 → 密集启动是均值回归,该反着做(向上破做空/向下破做多),
  挂限价 = maker 成本。
- **结果(三期 ALL)**:0.85/0.84/0.97 → 0.92/0.91/1.06(随参数);fade-long period3 到 1.29
  但 period2 0.63;fade-short period2 到 1.20 但 period1/3 塌。
- **机制**:仍是"一期/一边总在塌"的**同一互补翻转**;fade 比续势略好但净值 ~1.0。
- **关键推论**:**续势(IT-12)和 fade(IT-13)两方向都不赚 → 没有任何方向无关方式能提取这个动作。**
  加上逐 bar 方向=硬币(IT-02),**方向无关这扇门也关上了**。

### IT-12 · 突破双挂单 续势入场(方向无关)—— 🔴 淘汰(三期全亏)
- **假设**:密集=盘整,launch=突破;上沿 buy-stop/下沿 sell-stop,市场自己选方向,
  同时修 entry≠eye(等突破真启动才进)。taker 成本。
- **结果(三期 ALL)**:0.92/0.84/0.83(range12);两边都亏(L1517/S1727)。
- **机制**:突破后追进=**追高杀跌,买在顶卖在底**;这些密集启动**均值回归,非续势**。→ 推出 IT-13 fade。

### IT-11 · 策略动量自适应选边 —— 🔴 淘汰(最近期塌)
- **假设**:不用外部门,看策略自己近 K 天哪边在赚就做哪边(只用已平仓交易,causal)。
- **结果(两窗)**:K14 0.97/0.43;K21 **1.53**/0.72;K30 1.40/0.83。
- **机制**:regime 持续时(前期)追得上;**翻转段(最近期)追反**,越动量越亏。
- **→ 与 IT-10 合并出总结:方向/regime 选边 5 个角度全撞同一墙,病根是"翻转点当时不可知"。**

### IT-10 · causal regime 选边 + 各自判断层 —— 🔴 淘汰(打平)
- **假设**:IT-09 互补是真的,用 BTC 趋势 per-bar 选边、各配自己判断层,应能选对边。
- **结果(两窗)**:PF 0.997 / 1.02 —— 基本打平。
- **机制**:per-bar BTC 门太噪,把 IT-09 里"整期空头 1.67"那批好单切碎;**互补是多周级慢 regime,
  不是逐 bar**。IT-05 的失败在这里复现(换了独立判断层也没救回来)。

### IT-07 · 波动率 regime 门(H-A)—— 🔴 淘汰,但挖出"regime 会翻转"
- **假设**:空头在高波动 regime 有效(IT-06 里 btc_atr_pct/vol 主导)。
- **改动**:按 BTC atr_pct vs 其 500 根中位数分"高波动/低波动"两桶,各自 walk-forward。
- **结果(top20)**:
  - ALL:1.67 / 1.26 / **0.74**
  - 高波动:0.67 / 0.48 / **1.64**
  - 低波动:1.10 / **2.12** / 0.83
- **机制(重要)**:**没有任何波动桶三期都稳**。低波动空头前两期赚、period3 塌;高波动空头
  反过来。**"哪个 regime 有效"本身在随时间翻转**——这解释了为什么固定门救不了:规则在漂移。
- **→ 直接推出 IT-08 假设:静态模型跨 10 个月必然过时;需要滚动重训让判断层追着 regime 走。**

### IT-06 · 短周期做空质量特征 —— ✅ 保留(底座,但非终态)
- **假设**:period3 塌是因为判断层缺"这次跌破是真是假"的质量信号。
- **改动**:加 9 个空头质量特征(bd_depth/bd_force/bd_range/vol_break/ma_roll/lower_highs20/
  from_high20/room_to_low48/dd48)到做空判断层。
- **结果**:top10 = 1.92 / 0.89 / **1.12**(period3 从 0.88 转正)。
- **机制**:主导的却是 `btc_atr_pct`/`vol`/`rng` —— "空头何时有效"更取决于**市场波动 regime**,
  不是单根跌破的力度。**→ 下一条假设:显式波动率 regime。**

### IT-05 · regime 自适应方向(牛做多/熊做空)—— 🔴 淘汰(更差)
- **假设**:period3 该做多(long 0.93 > short 0.79),方向随 BTC-200均线 regime 切换。
- **结果**:整体被拉低,top10 = 0.92/0.84/**0.55**;做多那半(均线上方)本身亏(0.83)稀释了边。
- **教训**:BTC-200均线不是好方向判据;加做多稀释空头边。**固定做空 > 自适应。**

### IT-04 · 两层:v16 + 跌破全均线做空 + 判断层 —— ✅ 保留(IT-06 的底座)
- **结果**:top10 = 1.65/1.02/0.88;top20 period1 = **2.1**。判断层第一次真正提纯(非 hindsight)。
- **发现**:period3(0.88)是唯一没过的;raw 空头本身 1.21→0.77 衰减 = regime。

### IT-03 · 全均线突破方向规则(owner)—— ✅ 保留方向
- **规则**:收盘>全部6均线→多,<全部→空,中间跳过。TP5/SL2。
- **结果**:v16 火点 下方做空 **1.07**、上方做多 0.83;规则密集 短 1.03。
- **发现**:**空头是对的一边**;做多亏。这是第一个稳定 >1 的简单底座。

### IT-02 · 方向分类器(28+130+BTC+几何)—— 🔴 淘汰(方向不可测)

- **教训**:方向不能逐笔学 → 改用**规则方向(IT-03)+ regime**。

### IT-01 · 同源判断层(v16候选,预测 realized_ret,做多)—— 🔴 淘汰(regime luck)

- **教训**:①要分边 ②要 walk-forward ③做多是错的默认。

### IT-08 · 每月滚动重训(H-F)—— 🟡 不稳定(边是间歇性的)
- **假设**:regime 翻转 → 每月重训、追着当前 regime,守住每个月。
- **改动**:每个测试月 M,用全部 <M 数据训练,测 M 月,top20 空头。
- **结果(月度 top20 PF)**:09:1.13/10:1.44/11:1.23/12:0.65/01:1.17/02:1.84/03:0.64/04:1.40/05:0.26。
- **机制**:滚动重训也稳不住。**边是间歇性的**——约一半月份 ≥1.3、一半 <1,哪个月有效
  预测不了(regime 门 IT-07 + 滚动重训 IT-08 都试过)。~9 月净约 +35bps,勉强正、摆动巨大。
- **结论**:检测✓+方向✓(空)+判断✓(能提纯)都对,但**底层边本身间歇**,不是"常开"的
  可交易边。→ 下一步该正面问:能否**只在"边有效"的月识别并交易**(H-B 归因好月 vs 坏月)。
