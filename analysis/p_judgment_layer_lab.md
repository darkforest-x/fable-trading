## 1. 目标与成功标准

- **链路**:v16 检测(因果盘口)→ 方向规则 → **判断层过滤(本文件主攻)** → TP5/SL2 → 交易。
- **判断层的活 = 3 个决策**:①选点(top-decile)②方向(long/short)③regime(何时活跃)。
- **成功** = walk-forward 三期(尤其最近的 period3)top-decile PF@maker **稳健 ≥1.3**,
  不靠删样本、不靠单期独撑。达标 → 写预注册卡 → holdout#8。

**关键约定(避免自欺)**
- 只看**因果 PF**,不看 AUC/oracle(它们默认 hindsight)。
- 每个"救活"必须跨三期成立;单期漂亮(如 period1 2.1)不算通。
- 特征富堆已被证伪多次(28/116/130);新特征要有**机制假设**,不是撒网。

## 3. 迭代日志(倒序,最新在上)

### IT-07 · 波动率 regime 门(H-A)—— 🔴 淘汰,但挖出"regime 会翻转"
- **假设**:空头在高波动 regime 有效(IT-06 里 btc_atr_pct/vol 主导)。
- **改动**:按 BTC atr_pct vs 其 500 根中位数分"高波动/低波动"两桶,各自 walk-forward。
- **结果(top20)**:
  - ALL:1.67 / 1.26 / **0.74**
  - 高波动:0.67 / 0.48 / **1.64**
  - 低波动:1.10 / **2.12** / 0.83
- **机制(重要)**:**没有任何波动桶三期都稳**。低波动空头前两期赚、period3 塌;高波动空头
  反过来。**"哪个 regime 有效"本身在随时间翻转**——这解释了为什么固定门救不了:规则在漂移。
- **→ 直接推出 IT-08 假设:静态模型跨 10 个月必然过时;需要滚动重训让判断层追着 regime 走。**

### IT-06 · 短周期做空质量特征 —— ✅ 保留(底座,但非终态)
- **假设**:period3 塌是因为判断层缺"这次跌破是真是假"的质量信号。
- **改动**:加 9 个空头质量特征(bd_depth/bd_force/bd_range/vol_break/ma_roll/lower_highs20/
  from_high20/room_to_low48/dd48)到做空判断层。
- **结果**:top10 = 1.92 / 0.89 / **1.12**(period3 从 0.88 转正)。
- **机制**:主导的却是 `btc_atr_pct`/`vol`/`rng` —— "空头何时有效"更取决于**市场波动 regime**,
  不是单根跌破的力度。**→ 下一条假设:显式波动率 regime。**

### IT-05 · regime 自适应方向(牛做多/熊做空)—— 🔴 淘汰(更差)
- **假设**:period3 该做多(long 0.93 > short 0.79),方向随 BTC-200均线 regime 切换。
- **结果**:整体被拉低,top10 = 0.92/0.84/**0.55**;做多那半(均线上方)本身亏(0.83)稀释了边。
- **教训**:BTC-200均线不是好方向判据;加做多稀释空头边。**固定做空 > 自适应。**

### IT-04 · 两层:v16 + 跌破全均线做空 + 判断层 —— ✅ 保留(IT-06 的底座)
- **结果**:top10 = 1.65/1.02/0.88;top20 period1 = **2.1**。判断层第一次真正提纯(非 hindsight)。
- **发现**:period3(0.88)是唯一没过的;raw 空头本身 1.21→0.77 衰减 = regime。

### IT-03 · 全均线突破方向规则(owner)—— ✅ 保留方向
- **规则**:收盘>全部6均线→多,<全部→空,中间跳过。TP5/SL2。
- **结果**:v16 火点 下方做空 **1.07**、上方做多 0.83;规则密集 短 1.03。
- **发现**:**空头是对的一边**;做多亏。这是第一个稳定 >1 的简单底座。

### IT-02 · 方向分类器(28+130+BTC+几何)—— 🔴 淘汰(方向不可测)
- **结果**:方向 AUC 0.47-0.51 = 硬币;按预测方向交易靠 regime 蒙对。**逐笔方向预测不出来。**
- **教训**:方向不能逐笔学 → 改用**规则方向(IT-03)+ regime**。

### IT-01 · 同源判断层(v16候选,预测 realized_ret,做多)—— 🔴 淘汰(regime luck)
- **结果**:单次切分似正,walk-forward 拆穿(0.64/0.85/1.02),全做多默认错方向。
- **教训**:①要分边 ②要 walk-forward ③做多是错的默认。

### IT-00 · 基线:v16 + 全做多 + TP5/SL2 —— 🔴 holdout#6 否决
- **结果**:holdout PF 0.78。全做多 + 判断层同源仍反预测。**这是重构的起点。**

### IT-08 · 每月滚动重训(H-F)—— 🟡 不稳定(边是间歇性的)
- **假设**:regime 翻转 → 每月重训、追着当前 regime,守住每个月。
- **改动**:每个测试月 M,用全部 <M 数据训练,测 M 月,top20 空头。
- **结果(月度 top20 PF)**:09:1.13/10:1.44/11:1.23/12:0.65/01:1.17/02:1.84/03:0.64/04:1.40/05:0.26。
- **机制**:滚动重训也稳不住。**边是间歇性的**——约一半月份 ≥1.3、一半 <1,哪个月有效
  预测不了(regime 门 IT-07 + 滚动重训 IT-08 都试过)。~9 月净约 +35bps,勉强正、摆动巨大。
- **结论**:检测✓+方向✓(空)+判断✓(能提纯)都对,但**底层边本身间歇**,不是"常开"的
  可交易边。→ 下一步该正面问:能否**只在"边有效"的月识别并交易**(H-B 归因好月 vs 坏月)。
