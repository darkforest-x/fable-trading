# 项目路线图（三阶段）

原则：每阶段有明确验收标准，不通过不进入下一阶段；单变量实验纪律；极简实现。

**架构决定（项目所有者，2026-07-07）**：保留 YOLO，采用两阶段架构——
"检测层"（YOLO 找双均线密集区域）+ "判断层"（数值特征分类器判断是否会真正启动）。
检测层解决旧项目"固定框假检测"的问题；判断层解决"图形好看 ≠ 会涨"的问题。

## 阶段 1：P0 alpha 检验（已完成，2026-07-07）

**目标**：回答"人工标注的'完美启动结构'正样本，其未来收益分布是否显著优于同规则预筛的负样本"。

**结论**（详见 `analysis/p0_alpha_report.md`）：

- 收益端（future_favorable_pct，MFE）：**无 alpha**。合并去重 301 正 / 1365 负，
  Mann-Whitney p = 0.844，rank-biserial r = −0.007，正负样本中位 MFE 几乎相同（1.79% vs 1.73%）；
- 风险端（future_adverse_pct，MAE）：**有显著 alpha**。p ≈ 9.2e-14，r = +0.274，
  正样本中位回撤 −0.39% vs 负样本 −1.19%；
- 判定：人工标签识别的是"跌不动的点"而非"会涨的点"，**不能直接当判断层的分类目标**。

**对阶段 2 的直接影响**：判断层标签采用 **triple-barrier 结果导向标注**（P0 无收益端 alpha 触发的路线）；
人工标签降级为辅助过滤/特征。

## 阶段 2：两层架构

### 2a 检测层：YOLO 检测双均线密集区域

- **标注全自动、规则化**：按明确的均线间距阈值（如 fast_spread / full_spread 上限）在
  历史 K 线图上自动生成标注框，框的位置和宽度随真实密集区变化（不再是固定右缘竖条）；
- 样本量可达数万（旧项目合成数据 mAP50 0.98 已证明 YOLO 能学会这类几何模式）；
- 关闭 fliplr/mosaic/hsv 等破坏时间方向与红绿语义的增强；
- 检测层只回答"哪里有密集区"，**不做任何涨跌判断**。

**验收标准（2a）**：

- 真实数据验证集 mAP50 ≥ 0.90，且与纯规则扫描（同阈值直接算指标）对照，检出一致率 ≥ 95%——
  YOLO 在这里是规则的视觉替身，一致率低说明标注生成或训练有 bug；
- 验证集按时间切分，禁止随机切分。

### 2b 判断层：密集区域 → "是否会真正启动"分类器

- 对检测层输出的每个密集区域，从 OHLCV 直接计算数值特征：均线间距序列（收敛速度/形态）、
  密集持续根数、量比/量能 z 分数、波动率位置、与慢速均线的相对位置等
  （P0 已验证 ma_spread_pct 与未来 MFE 的 Spearman ρ≈0.4，特征路线有依据）；
- 轻量分类器（LightGBM 起步）；
- **标签：triple-barrier**（止盈/止损/超时，标签 = 先触哪个障碍），直接对齐交易盈亏；
  这是 P0 结论决定的——人工标签无收益端 alpha，不能当主目标。
  人工标签可作为可选辅助特征或风控叠加（其"低回撤"信息真实存在），须单变量验证增益；
- 按时间切分训练/验证，禁止随机切分；特征与标签严格无前视。

**验收标准（2b）**：

- 时间外验证集 AUC 显著 > 0.5（置换检验 p < 0.01）；
- top-decile 信号的 triple-barrier 期望收益扣除 0.2% 往返成本后为正；
- 每次特征/参数改动记入实验日志（单变量纪律）。

## 阶段 3：回测框架 + 冻结 holdout + 实验纪律

**目标**：把"检测层 + 判断层"整条流水线放进带成本的事件驱动回测，得到可信的样本外绩效。

**做法**：

- 简单事件驱动回测（taker 费 + 滑点 + 资金费近似），检测 → 判断 → 持仓 → 平仓全链路；
- **冻结 holdout**：留出最近 N 个月数据，全程不参与检测层标注阈值调整、判断层调参，
  只在最终验收时跑一次；
- **单变量实验纪律**：每次只改一个变量（一个特征、一个阈值、一个参数），
  实验记录（假设/改动/结果）追加到实验日志，无论成败都保留。

**验收标准**：

- holdout 上（仅跑一次）：扣费后收益为正，PF ≥ 1.3，最大回撤 ≤ 20%，交易数 ≥ 100 笔；
- 不满足则回到阶段 2，禁止在 holdout 上迭代。
