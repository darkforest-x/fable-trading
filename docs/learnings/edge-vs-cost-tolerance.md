# 单笔 edge 与成本同数量级时，回测结论是成本假设的函数

- **问题**：阶段 3 回测，同一策略在 0.2%/0.3%/0.4% 三档往返成本下 PF 分别为
  1.30 / 1.01 / 0.79——从"接近达标"到"亏损"只隔两档假设。
- **死胡同**：想用"选个合理的成本数"一锤定音。不存在这个数：taker/maker、
  币种流动性、单量大小都会让真实成本在这个区间内浮动。
- **有效路径**：把敏感度扫描当作结论本体而不是附录——正确的表述不是
  "策略 PF 1.01"，而是"策略的盈亏平衡成本 ≈ 0.30%，当前 edge 对执行质量
  没有安全边际"。这直接把下一步从"调参"改写成"降成本（maker 执行）或
  增大单笔毛利（标签结构）"两条结构性路线。
- **通用规则**：回测验收前先算盈亏平衡成本（mean gross per trade），
  与可实现成本区间比较：若安全边际 < 2 倍，任何单点成本假设下的
  验收结论都不可靠，必须报告敏感度并按结构性方案迭代。
  另：模型 in-sample 期的漂亮 PF（本例 1.82 vs 样本外 1.01）永远不作证据。
