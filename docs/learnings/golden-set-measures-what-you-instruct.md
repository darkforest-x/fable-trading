# 金标准测的是"你给的判据"，不是你以为的概念

- **问题**：owner 按"会不会想埋伏"标注 80 张，删掉规则框的 70%。这到底证明
  规则误标率 70%，还是证明检测层概念（几何密集）≠交易概念（值得埋伏）？
- **死胡同**：直接把 70% 当规则错误率去改阈值——网格扫描证明无论怎么调，
  规则家族贴合 owner 眼睛的 F1 天花板只有 ~0.45：这不是参数问题，是表达力问题。
- **有效路径**：承认判据即定义——owner 的口径成为检测层新目标后，正确路线是
  用 owner 标注直接训练（规则降级为高召回候选生成器），而不是继续拧阈值。
- **通用规则**：设计人工标注时，标注指令本身就是被测量的概念定义，写指令前先
  确认"这测的是哪一层的概念"；发现分歧率巨大时，先做表达力上限测试
  （网格拟合金标准）再决定修规则还是换范式。二轮标注应掺入一轮重复样本，
  同时测量标注者自一致性。

  scripts/fit_rules_to_golden.py。
