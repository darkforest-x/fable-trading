# "有信号没利润"的病根在标签经济学，不在模型

- **问题**：2b-v1 holdout AUC 0.591（p=0.002，信号真实）但 top-decile 扣 0.2% 成本
  净收益 -0.11%。模型对了，账算不平。
- **死胡同**：直觉反应是加特征、调参、换模型——全都没用，因为问题不在预测精度。
- **有效路径**：算成本占比——密集点天然低波动，v1 的 TP=2×ATR 毛利只有 ~0.5%，
  成本吃掉 40%+。v2 把 TP 加宽到 4×ATR（毛利 ~1.1%，成本占比 <20%）后，
  同样的特征和模型，top-decile 净收益转正（+0.09%~+0.10%）。
- **通用规则**：分类指标（AUC）和交易盈亏之间隔着一层"标签的收益尺度"。
  模型迭代前先算：标签为正时的中位毛利是往返成本的几倍？低于 3 倍就先修标签，别动模型。
  代价要心里有数：障碍加宽后任务更难，AUC 会降（0.59 → 0.54-0.56），这是预期内的交换。
- **牵连**：`src/judgment/labeling.py`（TP_ATR_MULT/SL_ATR_MULT/ATR_PCT_MIN）；
  成本假设 0.2% 往返写在 PROJECT_PLAN，改它要 owner 批准。
