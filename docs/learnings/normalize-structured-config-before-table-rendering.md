# 结构化配置进入表格前必须先定义展示契约

- **问题**：实验注册表把字典配置直接插入表格，浏览器显示为 `[object Object]`；改成完整 JSON 后又把表格撑宽，关键指标不可见。
- **死胡同**：只把对象序列化为 JSON 能消除错误文本，但没有解决扫描效率和布局宽度，属于把数据正确性问题转成了界面问题。
- **有效路径**：在 API 边界优先提取配置 `name`，无名称时生成稳定的两项摘要；表格使用固定列宽与省略显示，原始配置仍留在详情接口。
- **通用规则**：异构 JSON 进入列表视图前，先明确“列表标签”和“原始详情”是两个契约，再用真实长数据做桌面与手机宽度验收。
- **牵连**：`src/webapp/experiment_registry.py`、`src/webapp/static/style.css`、`tests/test_ops_phase01.py`；需兼容仓库实际 Python 3.9 运行时。
