# 「能用」过滤器按整仓，不是 tip 唯一

- **问题**：扫描外源（wuzao topics）时，把「对本仓能拿来用」收成「对 tip 主线有没有解药」，可用清单被砍到 ≤8，判断/执行/看板辅图/运维/标注被误删或压成噪音。
- **死胡同**：用「聚焦主线」当借口做硬过滤——结果是报告对 Owner 原意答非所问；tip 是优先级，不是用途集合的外边界。
- **有效路径**：两列分离——（1）对本仓任一子系统是否有明确用法；（2）是否服务 tip（是/否/间接）只用于 A/B/C 排队。无 tip 解药 ≠ 从可用清单删除。
- **通用规则**：外源筛选先问「用在检测/判断/执行/看板/运维/数据/标注的哪一层」；再问「相对 tip 何时碰」。禁止用当前最高优先痛点当唯一准入门。
