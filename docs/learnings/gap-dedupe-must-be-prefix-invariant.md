# 时间序列候选去重必须保持前缀不变

- **问题**：前向扫描刷新数据后，三条已经记录的候选不再出现在当前候选集合；同一 q80
  账本因此出现“当前 10 个、累计 13 个”的矛盾。
- **死胡同**：把差异归因于阈值漂移或账本重复。模型、数据 hash 和 q80 阈值都未变化，
  13 条账本 key 也全部唯一。
- **有效路径**：对同一币种分别扫描旧数据前缀和追加未来 K 线后的完整数据。旧实现按全
  历史 `shape_score` 从高到低做 18 根去重，未来更高分候选会挤掉已经发生的早期候选；
  改为按时间从早到晚、最早候选先到先得后，前缀集合稳定。
- **通用规则**：无前视不仅要求单行特征只看过去，样本选择和去重也必须满足 prefix
  invariance：追加未来数据不能改变任何已结束前缀的输出。
- **牵连**：`src/judgment/candidates.py`、`src/judgment/forward_scan.py`、方向分类数据集和
  judgment 训练候选；旧冻结模型仍基于非因果去重，必须作为 challenger 重建，不能直接
  用本修复改写 ACTIVE 或主账本。
