# 路径型经济指标必须按信号时间计算

- **问题**：方向 manifest 为了生成分类目录按 class 分组，预测仍能一一对齐，但直接按
  manifest 行顺序计算 max drawdown 会把不同日期重新排列，得到不存在的资金曲线。
- **死胡同**：认为只要预测与行对齐，所有经济指标都与顺序无关。均值、胜率和 PF 基本
  无关，复利回撤明确依赖先后顺序。
- **有效路径**：保留预测与 manifest 的行对齐，在选出实际交易后按 `signal_time` 排序，
  再计算净收益序列和 max drawdown；红测用同一组三笔收益证明错误顺序把 5% 回撤算成
  9.75%。
- **通用规则**：分类指标按样本对齐，路径指标按事件时间对齐；同一评估器必须显式处理
  两种顺序，不能依赖上游文件恰好怎么排序。
- **牵连**：`src/judgment/direction_economics.py`、方向预测 CSV、固定成本经济评估；不改变
  任何收益值、成本假设或交易选择。
