# 高 AUC 不能替代交易经济性验收

- **问题**：P0-3 的 20/60/120 判断层对比里，SMA/EMA 20/60/120 的 val AUC
  明显高于 8-55（0.626 vs 0.560），但 top-decile 毛利和净收益显著更差。
- **死胡同**：看到 20/60/120 与 YOLO 检测层完全对齐，且 AUC 大幅提升，很容易把它
  当成主线候选直接替换；这会重复 v1 的错误，把分类能力误当作可交易收益。
- **有效路径**：同表同时放 AUC、置换 p、top-decile 毛利、扣成本净收益、胜率和
  单特征基线。排序时先看净收益是否过成本，再把 AUC 当辅助解释，而不是反过来。
- **通用规则**：任何判断层新特征或新候选池，如果出现“AUC 更高但净收益更差”，第一步
  不是救模型，而是承认它未通过收益主标准，并把“是否继续研究”升级给 owner。
- **牵连**：`src/judgment/candidates_v206.py`、`scripts/ma206_comparison.py`、
  `analysis/p2b_ma206_comparison.md`、成本假设 0.2% 与 maker0.06%。
