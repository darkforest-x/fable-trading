# OKX 公共 API：403 是 WAF 挑 UA，慢是延迟不是限速

- **问题**：`fetch_okx.py` 拉历史 K 线，先是全部 403，修好后预估 1 小时的任务实测要 10 小时。
- **死胡同**：403 时第一反应是"要 API key"或"IP 被封"——都不是。限速方向也想错了：
  瓶颈根本不在 OKX 的 20 req/2s 限制，串行时实际只跑到 0.6 req/s。
- **有效路径**：① 403 = OKX WAF 拒绝 python-urllib 默认 User-Agent，换浏览器 UA 即通；
  ② 慢 = 单请求往返延迟 ~1.6s（网络路径远），解法是按币种开线程池并行，
  用全局锁保证请求间隔 ≥0.12s（≤8.3 req/s，稳在官方限制内），10 小时 → ~1 小时。
- **通用规则**：遇到公共 API 403 先 curl 换 UA 试一次再谈鉴权；遇到"预估时间差一个量级"
  先 time 单请求，分清延迟瓶颈和限速瓶颈——前者用并发解决，后者才需要等。
- **牵连**：`src/data/fetch_okx.py`（REQUEST_HEADERS、_throttle、--workers）；
  断点续传靠 `.part.csv`，loader 的文件名正则天然忽略它们。
