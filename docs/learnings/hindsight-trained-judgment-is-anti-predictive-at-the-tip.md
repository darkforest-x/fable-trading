# 事后训练的判断层，在盘口是反预测的（分越高越亏）

- **问题**：v16 实时检测器纯检测在干净 holdout 上亏损（PF 0.78）。指望两层架构的
  判断层筛出盈利子集救回——结果判断层非但没救，**判断分越高越亏**（top5% PF 0.48
  < 全体 0.78）。
- **死胡同**：默认"两层架构里判断层总能提纯检测候选"。这个假设在**同分布**下成立，
  但当检测器换到一个全新分布（v16 盘口 tip）而判断层还是旧分布（v11 事后候选）训练
  的，判断层的"好"信号定义与新候选系统性错位——不只是失效，是**反向**。
- **有效路径**：不要只看"检测器好不好"或"判断层好不好"，要看**两层的训练分布是否
  同源**。v11 判断在"启动已打印"的事后候选上学到"像会涨"= 启动后的形态；v16 候选是
  "启动前"的盘口，高 v11 分 = "已经涨完的样子" = 无后续空间 → 挑中亏货。用回测记录的
  fire 逐笔复算判断分 vs 真实收益，一眼看出反相关（不用重跑检测）。
- **通用规则**：两层/多层流水线里，**上游一换分布，下游必须同步重标定**，否则下游从
  "提纯器"变成"反向选择器"。验证一个新检测器时，不能沿用旧判断层的阈值当"实盘会怎样"
  ——要么两层同源重训，要么先测纯检测边际。旧 PF 高很可能是两层在**同一个事后分布**上
  自洽的幻觉（检测看到未来、判断给未来形态高分），实时化后必然塌。
