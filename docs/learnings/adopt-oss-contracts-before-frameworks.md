# 开源项目先借契约再引框架

- **问题**：项目同时缺交易执行、实验治理、标注和调度能力，直接安装多个热门框架会扩大依赖面，却不一定修复最紧迫的证据缺口。
- **死胡同**：按 star 数迁移整套框架，或把更快的参数扫描误当成更可靠的研究。
- **有效路径**：先把缺口映射到成熟项目的具体契约，再用零依赖试点验证价值；本轮采用 DVC 的内容寻址模式修复冻结数据可复现性。
- **通用规则**：只有局部模式证明能改善正确性、可复现性或运维成本后，才评估引入完整依赖。
- **牵连**：`src/judgment/frozen.py`、冻结数据集、DVC、MLflow、Freqtrade、NautilusTrader。
